# 计划文档审查者提示模板

派发计划文档审查者子 Agent 时使用此模板。

**用途：** 验证计划块是否完整、是否与规格匹配，以及任务分解是否合理。

**派发时机：** 每个计划块编写完成后

```
Task tool (general-purpose):
  description: "审查计划块 N"
  prompt: |
    你是一名计划文档审查者。验证此计划块是否完整且已准备好实现。

    **待审查的计划块：** [PLAN_FILE_PATH] - 仅第 N 块
    **参考规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 要查看什么 |
    |------|-----------|
    | 完整性 | TODO 标记、占位符、未完成的任务、缺失的步骤 |
    | 规格对齐 | 块覆盖了相关规格需求，无范围蔓延 |
    | 任务分解 | 任务原子化、边界清晰、步骤可执行 |
    | 文件结构 | 文件有清晰的单一职责，按职责而非层级拆分 |
    | 文件大小 | 任何新建或修改的文件是否可能增长到难以整体推理的程度？ |
    | 任务语法 | 步骤上使用复选框语法（`- [ ]`）以便跟踪 |
    | 块大小 | 每块不超过 1000 行 |

    ## 关键检查

    重点检查：
    - 任何 TODO 标记或占位符文本
    - 内容写着"类似于 X"但没有实际内容的步骤
    - 不完整的任务定义
    - 缺失的验证步骤或预期输出
    - 计划持有多个职责或可能变得难以管理的文件

    ## 输出格式

    ## 计划审查 - 第 N 块

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [为何重要]

    **建议（仅供参考）：**
    - [不阻止批准的建议]
```

**审查者返回：** 状态、问题（如有）、建议
