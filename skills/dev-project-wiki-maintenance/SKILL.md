# 项目 Wiki 维护

## 概述

本技能负责将长期有效的项目知识写入 `wiki/`，并维护各层 `INDEX.md`。它只处理结论化知识，不处理执行流水。

**核心原则：** `docs` 记录过程，`wiki` 记录长期知识。

## 何时使用

在以下情况使用：

- 完成一个任务后，产生了可复用规则
- 形成了稳定结构、项目约定、功能认知或错误规避经验
- 修改了项目 skill 与项目流程的绑定关系
- 新增或调整 wiki 分类、页面、索引

## 不要使用于

- 单次执行日志
- 对话原文
- 临时尝试
- 尚未稳定的草稿结论

这些内容应保留在 `docs/executions/`，而不是 `wiki/`。

## 写入边界

| 内容类型 | 写入位置 |
|------|------|
| 执行日志、当前状态、阻塞 | `docs/executions/` |
| 某个功能本体知识 | `wiki/03-功能模块/` |
| Skill 的项目职责与边界 | `wiki/02-Skill体系/` |
| 跨功能复用经验 | `wiki/05-项目经验/` |
| 工具入口与集成规则 | `wiki/04-工具与集成/` |

### `03-功能模块` 与 `05-项目经验` 的边界

- 如果知识属于某个具体模块或功能本体，写入 `wiki/03-功能模块/`
- 如果知识可跨功能复用，写入 `wiki/05-项目经验/`

## 更新流程

1. 确定知识的目标目录。
2. 更新目标知识页面，写入结论化内容。
3. 更新该目录的 `INDEX.md`：
   - 子类别
   - 重要文档
   - 最近更新
4. 如果新增了新类别或新页面影响父层导航，再更新父级 `INDEX.md`。
5. 如果变更涉及项目 skill，额外更新 `wiki/02-Skill体系/skill-registry.md`。

## INDEX.md 规则

每层目录都必须有 `INDEX.md`，并固定包含：

- 本目录职责
- 子类别
- 重要文档
- 本层规则
- 最近更新

新增知识页面而不更新 `INDEX.md` 是不完整的 wiki 更新。

## 任务后规则

如果一个任务**确实**产生了长期有效知识，就必须更新 `wiki/`。

如果一个任务**没有**产生长期知识，不在本技能里补空页面，而是在 `docs/executions/CURRENT.md` 中记录“本任务无 wiki 更新”。

## 红旗

**绝不：**

- 把执行日志写进 `wiki`
- 只改正文页面，不改 `INDEX.md`
- 把模块知识写进 `05-项目经验`
- 把跨功能经验写进某个模块页
- 在 skill 变更后忘记更新 `skill-registry.md`

## 集成

**由以下技能调用：**

- `dev:executing-plans`
- `dev:subagent-driven-development`

**经常一起更新：**

- `wiki/02-Skill体系/`
- `wiki/03-功能模块/`
- `wiki/05-项目经验/`
