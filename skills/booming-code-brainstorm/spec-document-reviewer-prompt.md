# 规格文档审查者提示模板

派发规格文档审查者子 Agent 时使用此模板。

**用途：** 验证规格是否完整、一致，并已准备好进行实现规划。

**派发时机：** 规格文档写入 docs/booming/<YYYY-MM-DD-HHmmss>-<feature-name>/specs/ 之后

```
Task tool (general-purpose):
  description: "审查规格文档"
  prompt: |
    你是一名规格文档审查者。验证此规格是否完整且已准备好规划。

    **待审查的规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别     | 要查看什么 |
    |----------|-----------|
    | 完整性   | TODO 标记、占位符、"TBD"、未完成的章节 |
    | 覆盖范围 | 缺失的错误处理、边缘情况、集成点 |
    | 一致性   | 内部矛盾、冲突的需求 |
    | 清晰性   | 模糊的需求 |
    | YAGNI    | 未被请求的功能、过度设计 |
    | 范围     | 足够专注于单一计划——不涵盖多个独立子系统 |
    | 架构     | 具有清晰边界、定义良好接口、可独立理解和测试的单元 |

    ## 关键检查

    重点检查：
    - 任何 TODO 标记或占位符文本
    - 写着"稍后定义"或"当 X 完成时再规格"的章节
    - 明显比其他章节详细度低的章节
    - 缺乏清晰边界或接口的单元——你能不阅读其内部实现就理解每个单元的功能吗？

    ## 输出格式

    ## 规格审查

    **状态：** ✅ 已批准 | ❌ 发现问题

    **问题（如有）：**
    - [章节 X]：[具体问题] - [为何重要]

    **建议（仅供参考）：**
    - [不阻止批准的建议]
```

**审查者返回：** 状态、问题（如有）、建议
